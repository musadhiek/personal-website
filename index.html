Data Structure and Algorithms 101

Intro To master software development you have to master Data Structure and Algorithms. To help you with mastering these topics I designed the Data Structure and Algorithms seminar series. The first part that is Data Structure and Algorithms 101 will give a brief introduction to the topics. Here we won't discuss the topics in detail but it will be done in the upcoming seminars. Let’s start with data structure, the most important task of the software is to handle the data given to it efficiently while performing the operations like adding,deleting,searching and sorting . if the data stored in the db is not of any particular order then performing these tasks will become painfully time consuming . To solve this we have to store data in a structured manner so that the system can perform the operations efficiently .So we can say that data structure is a particular way of organizing the data so that it can be used effectively . There are multiple types of data structures used in software development. Basic Data structure can be classified as below 1-Array An array is a collection of items stored at contiguous memory locations. The idea is to store multiple items of the same type together 2-Linked List Linked list data structure is linked together by references or pointers 2-Stack Stack can be logically thought of as a physical stack or pile of books or similar items where we can add or remove items from top of the stack only.it is known as first in last out structure 3-Queues Queue is similar to a real world queue in railway stations or any other place where people are joined from the back or tail and people leave from the front or head . this type of structure is known as first in first out 4-Tree Tree can be said as a group of nodes in which a parent child relation is kept. Here each node is a data structure containing a value and a reference to the adjacent child node.here the constraints are that the node will have only references to its childs and not to it’s parent if any , also no reference should be duplicated.
